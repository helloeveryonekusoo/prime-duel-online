# PRIME DUEL

3枚までの数字カードを合計して素数を作り、相手のライフカードを奪う2人用オンライン対戦カードゲームです。ホストが作った6文字のルームコードを使い、別々の端末から参加できます。通信はWebSocket対応MQTTブローカーを通して同期します。

## 遊び方

- 攻撃側は手札から1〜3枚を選び、選んだカードの数字の合計で素数を作ります。合計値が基本攻撃値になります。
- 7〜13のカードは1枚まで、4〜6のカードも1枚まで選択できます。1〜3には数字帯制限がありません。
- 防御側は手札1〜2枚の数字の合計、または共有の外部ゾーンから防御します。
- 攻撃値と防御値の差がダメージ枚数になります。残っているライフカードのうち、番号が小さいものからダメージ枚数分を防御側の手札へ移します。ライフゾーンが0枚になったら敗北です。
- 防御とダメージ処理の結果を表示した後、ボタン操作なしで自動的に次のターンへ進みます。
- Aは使用枚数分ドロー、Bは最終攻撃値+1、Cは外部ゾーンの防御値-2です。
- 攻撃はいつでもパスできます。パスしたプレイヤーは次の自分のターン、通常ドローの代わりに山札の上2枚から1枚を選んで引きます。
- 手札の上限は13枚です。14枚以上になった場合、13枚になるまで超過分のカードを選んで墓地へ送ります。

## 起動

公開済みのGitHub Pagesを、対戦する両方の端末で開いてください。ローカルで確認する場合は、ES Modulesを読み込める静的HTTPサーバーを使用します。

## ルーム対戦

1. 片方のプレイヤーが名前を入力して「ルームを作る」を押します。
2. 表示された6文字のコードを相手へ伝えます。
3. 相手は名前とコードを入力して「ルームに参加」を押します。
4. 2人がそろうと自動的にゲームが始まります。
5. ゲーム終了後、両プレイヤーが「同じ相手と再戦」を押すと、ルームを作り直さず次のゲームを開始できます。

## 観戦

ルームコードを入力して「観戦する」を押すと、進行中の対戦を観戦できます。観戦者には両プレイヤーの手札が表向きで表示されますが、ゲーム操作はできません。対戦開始前から待機することも、対戦途中から参加することもできます。対戦中のプレイヤー同士には、相手の手札は公開されません。

対戦画面では両プレイヤーの墓地を開いて、使用済みカードの数字を確認できます。ゲームログは更新時に最新の動作まで自動スクロールします。

プレイヤーは自分の墓地では数字とA/B/Cを確認できますが、相手の墓地では数字だけが表示されます。防御時の画面にはB効果の加算内訳を表示せず、確定した攻撃値だけを表示します。

## 公開

GitHub Pages上でそのまま動作します。[MQTT.js](https://github.com/mqttjs/MQTT.js)とHiveMQの公開ブローカーを使用し、ルーム探索とゲームデータをMQTT over WebSocketで同期します。WebRTCは使用しないため、端末や回線による直接接続の相性問題を避けられます。ルームコードは信頼できる相手とのみ共有してください。
